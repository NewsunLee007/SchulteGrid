# Schulte Grid (舒尔特方格) 生成器实现计划

## 1. 项目结构与基础设置
- 创建三个核心文件：`index.html`, `styles.css`, `script.js`，保持代码结构清晰可维护。
- 设置 HTML5 基础结构，配置 `viewport` 元标签以完美支持移动端竖屏响应式显示。

## 2. 核心 UI/UX 设计 (Mobile-First)
- **视觉风格 (Aesthetic)**: 采用极简主义 (Minimalist) 风格。界面以高对比度、清晰易读为主，排版精美。剔除多余的视觉干扰，专注于“专注力训练”的核心目标。
- **控制面板 (Controls)**: 
  - 界面顶部或底部设置悬浮控制区，适配手机单手操作。
  - **网格设置**: 提供滑动条或下拉菜单，允许用户自由设置方格数量（如 $3 \times 3$ 到 $10 \times 10$）。
  - **操作按钮**: 提供“重新生成”按钮和“打印 / 保存为 PDF”按钮。
- **方格展示区 (Grid Display)**: 
  - 屏幕居中显示，使用 CSS Grid 动态布局。
  - 格子保持严格的正方形（Aspect Ratio），数字字体大小基于屏幕宽度和格子数量进行动态计算 (`clamp` 或 `calc`)，确保在任何竖屏设备上都饱满美观。

## 3. 核心交互逻辑 (JavaScript)
- **核心算法**: 编写高效的 Fisher-Yates 洗牌算法，根据选定的阶数 $N$ 生成 $1$ 到 $N^2$ 的随机不重复数字序列。
- **动态渲染**: 根据用户的设置和随机序列，动态清空并重建 DOM 中的网格元素。
- **交互绑定**: 监听设置变更以实时更新网格；监听“打印”按钮以调用浏览器的原生打印 API (`window.print()`)，这是生成高质量 PDF 的最佳方式。

## 4. 打印与 PDF 导出排版优化 (CSS `@media print`)
这是满足“直接打印”和“A4 大小排版”的核心：
- **页面尺寸规范**: 使用 `@page { size: A4 portrait; margin: 20mm; }` 确保默认导出为 A4 竖版尺寸。
- **元素显隐控制**: 打印时自动隐藏所有的 UI 控制面板、按钮、背景阴影等非核心元素。
- **黑白高对比度打印**: 将网格强制转换为纯黑边框和纯黑字体，移除所有可能浪费墨水或影响打印清晰度的背景色。
- **打印版面布局**: 确保生成的舒尔特方格在 A4 纸张上垂直且水平居中，尺寸自动撑满打印安全区域，并在顶部自动添加“舒尔特方格训练”等打印标题。

## 5. 最终检查
- 检查移动端竖屏浏览器的显示效果。
- 检查桌面端浏览器打印预览中的 PDF 生成效果，确保严格符合 A4 纸排版标准。
